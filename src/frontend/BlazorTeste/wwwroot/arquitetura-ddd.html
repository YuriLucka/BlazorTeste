Fluxo de Autenticação JWT
1
Login: POST /api/auth/login → retorna { accessToken, nome, email }. API define cookie HttpOnly com refresh token.
2
Token em memória: TokenService armazena JWT em memória (não localStorage). Cada requisição injeta Authorization: Bearer {jwt}.
3
Refresh automático: ao carregar app ou receber 401, TokenService chama POST /api/auth/refresh usando cookie. Novo JWT emitido sem re-login.
4
Logout: POST /api/auth/logout → API revoga cookie. Frontend limpa EntidadeAtiva e redireciona para /login.
Isolamento de camadas: o Blazor WASM nunca acessa diretamente EF Core, SQL Server ou classes de Domain/Application. Todo acesso é via API REST com JWT. A API valida o token em cada requisição antes de acionar os AppServices.
